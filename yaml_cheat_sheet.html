YAML Cheat Sheet

Basic Data Types

YAML supports various data types, including strings, numbers, and booleans.


# Strings - Can be enclosed in double or single quotes.
name: "John Doe"
city: 'New York'
multiline: |
  This is a multiline
  string in YAML.
multiline_folded: >
  This is a folded multiline
  string in YAML that removes
  newlines and replaces them with spaces.
        

# Numbers - Can be integers or floating-point values.
age: 30
height: 5.9
        

# Booleans - Represent true or false values.
is_active: true
is_admin: false

Complex Data Types

YAML supports structured data like lists and dictionaries.


# Lists - Represent sequences of values.
fruits:
  - Apple
  - Banana
  - Cherry
        

# Dictionaries - Key-value pairs representing objects.
person:
  name: Alice
  age: 28
  skills:
    - Python
    - YAML

Advanced YAML

Advanced features include nested structures, anchors, aliases, and merge keys.


# Nested Structures - YAML allows nesting of dictionaries and lists.
servers:
  - name: Server1
    ip: 192.168.1.1
  - name: Server2
    ip: 192.168.1.2
        

# Anchors & Aliases - Define reusable values using '&' and reference them with '*'.
defaults: &defaults  # Define an anchor named 'defaults' with common settings
  timeout: 30
  retries: 5

service1:
  <<: *defaults  # Merge the 'defaults' values into 'service1'
  name: API Service  # Override or add additional properties
        

# Merge Keys - Extend or inherit configurations from another key.
base: &base  # Define an anchor named 'base' with common settings
  name: Common Config
  debug: true

development:
  <<: *base  # Merge the 'base' values into 'development'
  environment: dev  # Override or add additional properties
